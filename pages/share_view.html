Essential optimization techniques for React applications including memo, useMemo, and useCallback best practices that can dramatically improve your app's performance.

1. React.memo for Component Optimization

React.memo is a higher-order component that prevents unnecessary re-renders by memoizing the result. Use it when your component renders the same result given the same props.

const MyComponent = React.memo(({ name, age }) => {
  return (
    <div>
      <h3>{name}</h3>
      <p>Age: {age}</p>
    </div>
  );
});

// Only re-renders when name or age changes
export default MyComponent;

2. useMemo for Expensive Calculations

useMemo helps you avoid expensive calculations on every render by memoizing the result until dependencies change.

const ExpensiveComponent = ({ items }) => {
  const expensiveValue = useMemo(() => {
    return items.reduce((sum, item) => sum + item.value, 0);
  }, [items]);

  return <div>Total: {expensiveValue}</div>;
};

3. useCallback for Function Memoization

useCallback returns a memoized version of the callback that only changes if dependencies change. Perfect for preventing child component re-renders.

const ParentComponent = ({ items }) => {
  const [filter, setFilter] = useState('');

  const handleItemClick = useCallback((id) => {
    // Handle item click logic
    console.log('Item clicked:', id);
  }, []);

  return (
    <div>
      {items.map(item => (
        <ChildComponent 
          key={item.id}
          item={item}
          onClick={handleItemClick}
        />
      ))}
    </div>
  );
};

4. Key Performance Tips

  • Use React DevTools Profiler to identify performance bottlenecks
  • Avoid creating objects and functions in render methods
  • Implement proper key props for list items
  • Consider code splitting with React.lazy and Suspense
  • Use production builds for performance testing

Pro Tip: Don't optimize prematurely. Profile your app first to identify actual performance issues before applying these techniques.

Remember that these optimization techniques should be used judiciously. Over-optimization can sometimes hurt performance more than help it. Always measure the impact of your optimizations.
